  **deep/models.py**: Data models for deep validation results:

  ```python
  from dataclasses import dataclass, asdict
  from enum import Enum

  class DeepCheckType(Enum):
      SIGNATURE_COMPLETE = "signature_complete"    # All params in signature have :param: docs
      EXAMPLES_NONTRIVIAL = "examples_nontrivial"  # Examples section has code blocks with >1 line
      RETURN_TYPE_DOCUMENTED = "return_type_documented"  # Has :return: or :rtype: field
      SEEALSO_PRESENT = "seealso_present"          # Has a .. seealso:: directive

  @dataclass
  class DeepCheckResult:
      check: DeepCheckType
      passed: bool
      detail: str  # Human-readable explanation ("3 of 3 params documented" or "Missing :param x:")

  @dataclass
  class DeepFunctionResult:
      function_name: str
      doc_page: str           # e.g. "command_ref/ols"
      file_path: str          # absolute path to RST file
      checks: list[DeepCheckResult]
      overall_pass: bool      # True only if ALL checks passed

      def to_dict(self) -> dict:
          d = asdict(self)
          d["checks"] = [{"check": c.check.value, "passed": c.passed, "detail": c.detail} for c in self.checks]
          return d
  ```

  **deep/checker.py**: `deep_check_function(parsed_doc: ParsedDoc) -> list[DeepCheckResult]` that runs 4 checks:

  1. **SIGNATURE_COMPLETE**: Reuse pattern from SignatureChecker._signature_has_args(). Extract param names from function directive signature (e.g., `.. function:: ols(y, x)` -> ["y", "x"]). Check that every param has a matching `:param name:` field in parsed_doc.field_lists. Pass if all params documented. Detail: "3/3 params documented" or "Missing: x, z".

  2. **EXAMPLES_NONTRIVIAL**: Check parsed_doc.code_blocks for non-empty blocks with content length > 20 chars (not just `x = ols(y, x);`). Pass if at least one code block has substantive content. Detail: "2 code blocks, longest is 5 lines" or "No code blocks found" or "Only trivial examples (< 20 chars)".

  3. **RETURN_TYPE_DOCUMENTED**: Check parsed_doc.field_lists for any entry where name starts with "return" or "rtype". Pass if found. Detail: "Has :rtype: field" or "No return type documentation".

  4. **SEEALSO_PRESENT**: Read raw RST content from parsed_doc.path, search for `.. seealso::` directive using the same SEEALSO_RE pattern from checkers/seealso.py. Pass if found. Detail: "See Also section present" or "No See Also section".

  Also create `deep_check_functions(target_names, docs_dir, env) -> list[DeepFunctionResult]`:
  - Takes list of function names (from frequency target list or --top-n)
  - Looks up each function in env.domaindata["gauss"]["objects"] to get docname
  - Resolves docname to RST file path: `docs_dir / (docname + ".rst")`
  - Parses with parse_rst(filepath, content, DocType.COMMAND_REF)
  - Runs deep_check_function() on each
  - Returns list of DeepFunctionResult with overall_pass = all(c.passed for c in checks)

  **tests/test_deep_checker.py**: Unit tests using synthetic RST content (same fixture pattern as test_sections.py and test_signatures.py):

  - Test function with all 4 checks passing (complete signature, examples, return type, seealso)
  - Test function missing params -> SIGNATURE_COMPLETE fails
  - Test function with empty/trivial example -> EXAMPLES_NONTRIVIAL fails
  - Test function without return type -> RETURN_TYPE_DOCUMENTED fails
  - Test function without seealso -> SEEALSO_PRESENT fails
  - Test overall_pass is False when any check fails
