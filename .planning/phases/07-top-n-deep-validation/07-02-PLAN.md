  **Step 1: Add AI check type to models.py**

  Add a new DeepCheckType enum value:
  ```python
  AI_EXAMPLE_QUALITY = "ai_example_quality"  # AI verification of example code correctness
  ```

  **Step 2: Create deep/ai_checker.py**

  Create a module that uses the Claude API (following the same lazy-import pattern as ai/reviewer.py) to verify example code correctness for a single function page.

  `ai_check_examples(parsed_doc: ParsedDoc, function_name: str) -> DeepCheckResult`:

  1. Extract all code blocks from parsed_doc.code_blocks (skip empty ones).
  2. If no code blocks exist, return DeepCheckResult(AI_EXAMPLE_QUALITY, passed=True, detail="No examples to verify").
  3. Build a focused system prompt:
     ```
     You are a GAUSS programming language expert reviewing documentation examples.
     Check the example code for the function '{function_name}'.

     Flag issues ONLY if you are confident they are real problems:
     1. Example calls a different function than the one being documented
     2. Parameters that are clearly wrong (wrong type, impossible values)
     3. Comments that contradict what the code does
     4. Code that would produce an error in GAUSS

     Do NOT flag:
     - Style preferences
     - Missing error handling
     - Simplistic examples (that's OK for documentation)
     ```
  4. Send the code blocks as user message content.
  5. Use Pydantic structured output with a new schema:
     ```python
     class ExampleCheckResult(BaseModel):
         has_issues: bool
         issues: list[str]  # Each issue is a one-line description
     ```
  6. Return DeepCheckResult:
     - passed = not has_issues
     - detail = "No issues found" if passed, else "; ".join(issues)

  Use model "claude-sonnet-4-20250514" (same as existing personas).
  Use temperature=0 for deterministic output.
  Lazy import `anthropic` at call time (same pattern as reviewer.py).

  Also create `ai_check_examples_batch(results: list[DeepFunctionResult], docs_dir: str) -> list[DeepFunctionResult]`:
  - Takes the structural check results from deep_check_functions()
  - For each result, reads the RST file, parses it, runs ai_check_examples()
  - Appends the AI DeepCheckResult to result.checks
  - Updates result.overall_pass to include the AI check
  - Returns the updated list

  **Step 3: Create tests/test_deep_ai_checker.py**

  Unit tests with mocked Claude API (mock anthropic.Anthropic().messages.parse):

  - Test ai_check_examples with no issues found -> passed=True
  - Test ai_check_examples with issues found -> passed=False, detail contains issue text
  - Test ai_check_examples with no code blocks -> passed=True, detail="No examples to verify"
  - Test ai_check_examples_batch updates overall_pass correctly

  **Add `deep-validate` subcommand to cli.py:**

  ```python
  @cli.command("deep-validate")
  @click.option("--top-n", type=int, default=100,
                help="Number of top functions to validate (default: 100)")
  @click.option("--targets-file", type=click.Path(exists=True), default=None,
                help="File with function names (one per line), e.g., from freq --output-targets")
  @click.option("--no-ai", is_flag=True, default=False,
                help="Skip AI-assisted example verification (faster, structural checks only)")
  @click.option("--no-blog", is_flag=True, default=False,
                help="Skip blog scraping when computing frequency ranking")
  @click.option("--format", "output_format",
                type=click.Choice(["terminal", "json", "markdown"]),
                default="terminal", help="Output format")
  @click.option("--output", "-o", type=click.Path(), default=None,
                help="Write report to file instead of stdout")
  @click.pass_context
  def deep_validate(ctx, top_n, targets_file, no_ai, no_blog, output_format, output):
      """Deep validation of top-N function pages."""
  ```

  Pipeline logic:

  1. **Determine target function names:**
     - If `--targets-file` provided: read file, one name per line, strip whitespace, skip blank lines
     - Else: run frequency ranking pipeline inline (same logic as `freq` command):
       - Load Sphinx env
       - Count crossrefs
       - Optionally scrape blog (unless --no-blog)
       - rank_functions()
       - Take top `top_n` names

  2. **Load Sphinx environment** (lazy import, same pattern):
     ```python
     from gauss_doc_qa.parser.sphinx_env import load_sphinx_env
     env = load_sphinx_env(docs_dir)
     ```

  3. **Run structural deep checks:**
     ```python
     from gauss_doc_qa.deep.checker import deep_check_functions
     results = deep_check_functions(target_names, docs_dir, env)
     ```

  4. **Run AI checks (unless --no-ai):**
     ```python
     if not no_ai:
         import os
         if not os.environ.get("ANTHROPIC_API_KEY"):
             click.echo("Warning: ANTHROPIC_API_KEY not set, skipping AI checks. Use --no-ai to suppress this warning.")
         else:
             from gauss_doc_qa.deep.ai_checker import ai_check_examples_batch
             results = ai_check_examples_batch(results, docs_dir)
     ```

  5. **Render report:**
     ```python
     from gauss_doc_qa.deep.report import render_deep_terminal, render_deep_json, render_deep_markdown
     ```
     Follow same render pattern as freq command (terminal/json/markdown with optional --output file).

  6. **Summary line:**
     ```python
     passed = sum(1 for r in results if r.overall_pass)
     click.echo(f"Deep validation: {passed}/{len(results)} functions passed all checks")
     ```
