  **1. Update `fixer/__init__.py`:**
  Export the public API:
  ```python
  from gauss_doc_qa.fixer.models import FixProposal
  from gauss_doc_qa.fixer.resolver import resolve_fixes, resolve_func_ref
  ```

  **2. Add `fix` subcommand to `cli.py`:**

  ```python
  @cli.command()
  @click.option("--apply", is_flag=True, default=False,
                help="Apply fixes (default is dry-run)")
  @click.option("--verify", is_flag=True, default=False,
                help="Run Sphinx build verification after applying fixes")
  @click.option("--min-confidence", type=float, default=0.85,
                help="Minimum fuzzy match confidence for auto-fix")
  @click.pass_context
  def fix(ctx, apply, verify, min_confidence):
      """Auto-fix broken internal links (dry-run by default)."""
  ```

  Implementation flow:
  1. Load Sphinx environment (lazy import of load_sphinx_env, same pattern as check-refs).
  2. Parse all RST files (same scan_docs_dir + parse_rst pattern as scan command).
  3. Run LinksChecker and SeeAlsoChecker to get broken ref Findings.
  4. Extract gauss_objects from `env.domaindata['gauss']['objects']`.
  5. Call `resolve_fixes(findings, gauss_objects, min_confidence)` to get FixProposals.
  6. If no proposals: print "No auto-fixable issues found." and return.
  7. Call `apply_fixes(proposals, dry_run=not apply)` to get applied proposals.
  8. Render output:
     - For each applied proposal, print Rich-formatted diff:
       ```
       --- {file_path} (line {line_number})
       - {original_text}
       + {fixed_text}
         [confidence: {confidence:.2f}, category: {category}]
       ```
     - Use `rich.console.Console` with `style="red"` for old line, `style="green"` for new line.
     - Summary line: "Proposed fixes: {N} ({M} high confidence, {K} medium)" where high >= 0.95 and medium >= 0.85.
     - If dry-run: "Run with --apply to write changes."
     - If applied: "Applied {N} fixes."
  9. If `verify` and `apply`:
     - Print "Running Sphinx build verification..."
     - Call `verify_sphinx_build(docs_dir)`.
     - Print result: "Verification: {warning_count} warnings" with pass/fail indication.
     - If verify and not apply: print "Skipping --verify (no changes applied)."
  </action>
