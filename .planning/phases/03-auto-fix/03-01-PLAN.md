  @dataclass
  class FixProposal:
      file_path: str
      line_number: int
      original_text: str      # the full line containing the broken ref
      fixed_text: str          # the line with corrected ref
      old_target: str          # e.g., "plotbar"

  **3. Create `fixer/resolver.py`:**

  Function `resolve_func_ref(broken_target: str, known_names: list[str], min_confidence: float = 0.85) -> tuple[str, float] | None`:
  - Use `difflib.get_close_matches()` with `n=3, cutoff=min_confidence` against casefolded known_names.
  - If exactly 1 match: return (original-cased name, confidence score).
  - If 2+ matches: compute ratios for top two. If top is better by >0.05, accept it. Otherwise return None (ambiguous).
  - If 0 matches: return None.
  - Map back from casefolded match to original casing via dict lookup.

  Function `resolve_fixes(findings: list[Finding], gauss_objects: dict, min_confidence: float = 0.85) -> list[FixProposal]`:
  - Filter findings to category in ("broken_func_ref", "broken_seealso_ref").
  - Extract broken target name from finding.message using regex: `r"'([^']+)'"`.
  - Call `resolve_func_ref()` for each.
  - For successful resolutions, read the source file at finding.file, get the line at finding.line, construct fixed_text by replacing the old target with new target inside the `:func:` role pattern (use `re.sub` with pattern `:func:`old_target`` -> `:func:`new_target``).
  - Return list of FixProposal objects.

  **1. Create `fixer/applier.py`:**

  Constants for safety checks:
  ```python
  TABLE_GRID_RE = re.compile(r"^\s*[+|][-=+|]+[+|]\s*$")
  TABLE_SIMPLE_RE = re.compile(r"^\s*[=]+(\s+[=]+)+\s*$")
  TABLE_CELL_RE = re.compile(r"^\s*\|")
  DIRECTIVE_RE = re.compile(r"^\s*\.\.\s+\w+")
  ```

  Function `is_safe_to_fix(line: str, line_num: int, all_lines: list[str], code_block_ranges: list[tuple[int, int]]) -> bool`:
  - Return False if line matches TABLE_GRID_RE or TABLE_SIMPLE_RE or TABLE_CELL_RE.
  - Return False if DIRECTIVE_RE matches AND "seealso" is NOT in the line (seealso content is safe).
  - Return False if line_num falls within any code_block_range (start_line, end_line).
  - Otherwise return True.
  - `code_block_ranges` is a list of (start, end) line number pairs for all literal blocks in the file.

  Helper `compute_code_block_ranges(lines: list[str]) -> list[tuple[int, int]]`:
  - Walk lines looking for `::` at end of line (after stripping) or `.. code-block::` directives.
  - After finding such a marker, a blank line follows, then all indented lines below form the code block.
  - Return list of (first_content_line, last_content_line) tuples.
  - Line numbers are 1-based to match Finding.line.

  Function `apply_fixes(proposals: list[FixProposal], dry_run: bool = True) -> list[FixProposal]`:
  - Group proposals by file_path.
  - For each file, read all lines.
  - Compute code_block_ranges from lines.
  - Sort proposals by line_number descending (bottom-up for safety).
  - For each proposal, call is_safe_to_fix(). If unsafe, skip it (log/print warning).
  - If safe and not dry_run: replace the line at proposal.line_number with proposal.fixed_text.
  - If not dry_run: write the modified lines back to file.
  - Return list of actually applied (or would-be-applied in dry-run) FixProposals.

  - Import `load_sphinx_env` from parser.sphinx_env.

  - Call `load_sphinx_env(docs_dir)`.

  - Return `{"success": bool, "warning_count": int, "warnings": list[str]}`.

  Implementation detail: The Sphinx constructor accepts a `warning` parameter. Pass `io.StringIO()` to capture warnings. After build, parse the StringIO content for warning count.
