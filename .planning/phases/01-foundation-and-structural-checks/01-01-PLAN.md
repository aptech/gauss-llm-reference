  Define these exact types:

  ```python
  from dataclasses import dataclass, asdict
  from enum import Enum

  class Severity(Enum):
      ERROR = "error"
      WARNING = "warning"
      INFO = "info"

  class DocType(Enum):
      COMMAND_REF = "command_ref"
      OPERATOR = "operator"
      ALPHA_INDEX = "alpha_index"
      GETTING_STARTED = "getting_started"
      USER_GUIDE = "user_guide"
      GRAPHICS_GUIDE = "graphics_guide"
      APP_MODULE = "app_module"
      INCLUDE_FRAGMENT = "include"
      OTHER = "other"

  @dataclass
  class CodeBlock:
      content: str
      line_number: int | None
      is_empty: bool

  @dataclass
  class Finding:
      file: str
      line: int | None
      severity: Severity
      category: str
      checker: str
      message: str
      auto_fixable: bool = False

      def to_dict(self) -> dict:
          d = asdict(self)
          d["severity"] = self.severity.value
          return d

  @dataclass
  class ParsedDoc:
      path: str
      doc_type: DocType
      title: str
      sections: list[str]          # normalized section titles (lowercase, stripped)
      code_blocks: list[CodeBlock]
      field_lists: list[dict]      # list of {name: str, body: str} from field_list nodes
      raw_doc: object              # docutils document node (for checkers that need deeper traversal)
  ```

  **parser/rst_parser.py:**
  Implement `parse_rst(filepath: str, content: str) -> ParsedDoc` using docutils:

  ```python
  from docutils.utils import new_document
  from docutils.parsers.rst import Parser
  from docutils.frontend import OptionParser
  from docutils import nodes
  from gauss_doc_qa.models import ParsedDoc, CodeBlock, DocType

  def parse_rst(filepath: str, content: str, doc_type: DocType = DocType.OTHER) -> ParsedDoc:
      parser = Parser()
      settings = OptionParser(components=(Parser,)).get_default_values()
      settings.report_level = 5  # SEVERE only -- suppress unknown directive warnings
      settings.halt_level = 5
      doc = new_document(filepath, settings)
      parser.parse(content, doc)

      title = _extract_title(doc)
      sections = _extract_sections(doc)
      code_blocks = _extract_code_blocks(doc)
      field_lists = _extract_field_lists(doc)

      return ParsedDoc(
          path=filepath,
          doc_type=doc_type,
          title=title,
          sections=sections,
          code_blocks=code_blocks,
          field_lists=field_lists,
          raw_doc=doc,
      )
  ```

  Helper functions:
  - `_extract_title(doc)` -- get first section's title child via `doc.traverse(nodes.section)`, return `""` if no sections
  - `_extract_sections(doc)` -- for each `nodes.section`, get title child text, normalize to lowercase stripped. Return `list[str]`.
  - `_extract_code_blocks(doc)` -- for each `nodes.literal_block`, create `CodeBlock(content=node.astext(), line_number=node.line, is_empty=not node.astext().strip())`
  - `_extract_field_lists(doc)` -- for each `nodes.field` in any `nodes.field_list`, extract `{"name": field_name.astext(), "body": field_body.astext()}`

  **Important:** The `line` attribute on literal_block nodes may be None for some constructs. Accept None.

  **parser/classifier.py:**
  Implement `classify_doc(filepath: str, base_dir: str) -> DocType`:

  Classification rules (in priority order):
  1. Path contains `/include/` segment -> `INCLUDE_FRAGMENT`
  2. Subdirectory mapping (check if any path component matches):
     - `getting-started` -> `GETTING_STARTED`
     - `user-guide` -> `USER_GUIDE`
     - `graphics-guide` -> `GRAPHICS_GUIDE`
     - Known app module dirs (tsmt, fanpac, cmlmt, comt, maxlikmt, garch, ldt, mgarch, msbvar, sslib, switch) -> `APP_MODULE`
  3. Top-level file, stem is single letter (a-z) or underscore `_` -> `ALPHA_INDEX`
  4. Top-level file, stem in known operator set -> `OPERATOR`
     Known operators: `addition`, `subtraction`, `multiplication`, `division`, `assignment`, `hat`, `dot-operators`, `kron`, `mod-operator`, `exponent`
  5. Default -> `COMMAND_REF`

  **Note:** The operator list above is a starting set. During implementation, read a few more files from ~/svn/gxmldoc/docs/ to identify any additional operator pages. The key identifier is: top-level RST file with Parameters/Returns sections instead of `.. function::` directive.
