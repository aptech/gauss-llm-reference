<objective>
Add glossary generation so users can auto-generate a draft glossary YAML from corpus term frequency analysis instead of manually curating one from scratch.

Purpose: The existing glossary system (Phase 5) requires a hand-written YAML file. For a large corpus, bootstrapping that file manually is tedious. This phase extracts frequent terms, groups variants, and outputs a draft YAML ready for human curation.
Output: `glossary-gen` CLI subcommand that produces a draft YAML glossary file.
</objective>

    1. `extract_terms(parsed_docs: list[ParsedDoc]) -> Counter`
       - Walk each ParsedDoc.raw_doc using docutils node traversal
       - Collect text from paragraph, title, and term nodes (nodes.paragraph, nodes.title, nodes.term)
       - SKIP text inside literal_block nodes (code examples) — check ancestors for literal_block
       - SKIP text inside system_message nodes (same pattern as rst_parser.py)
       - For each text chunk, extract:
         a. Multi-word terms: 2-3 word sequences where at least one word is capitalized and length >= 3 chars. Use regex: r'\b([A-Z][a-z]+(?:\s+[A-Za-z]{3,}){1,2})\b' to find capitalized-start phrases
         b. Single-word proper nouns: capitalized words >= 4 chars that appear capitalized >50% of the time (not just sentence-start). Use regex r'\b([A-Z][a-z]{3,})\b'
       - Maintain a stopword set: {"The", "This", "That", "These", "Those", "When", "Where", "Which", "What", "From", "Into", "With", "About", "After", "Before", "Between", "During", "Since", "Through", "Until", "Upon", "Also", "Both", "Each", "Either", "Every", "However", "Neither", "Other", "Some", "Such", "Than", "Then", "There", "Here", "Most", "More", "Less", "Many", "Much", "None", "Only", "Same", "Very", "Just", "Even", "Still", "Note", "Example", "Following", "Parameters", "Returns", "See", "Type", "Format", "Purpose", "Remarks", "Source", "Description", "Default", "Global", "Input", "Output", "Syntax"}
       - Return Counter with term -> total_count across all docs

    2. `group_terms(term_counts: Counter, min_freq: int = 3) -> list[dict]`
       - Filter to terms with count >= min_freq
       - Group terms by lowercase form (case variants become aliases)
       - Also group simple plural/singular: if "matrices" and "matrix" both exist, merge into one group (prefer singular as canonical)
       - For each group: canonical = most frequent variant; aliases = all other variants sorted alphabetically
       - Category = "auto-detected" (user will curate later)
       - Return list of dicts: [{"canonical": str, "aliases": [str], "category": str, "count": int}]
       - Sort output by count descending (most frequent terms first)

    3. `generate_glossary_yaml(groups: list[dict]) -> str`
       - Produce YAML string with top-level "glossary:" key
       - Each entry has: canonical, aliases, category, description (empty string placeholder)
       - Do NOT include the "count" field in YAML output (it's for sorting only)
       - Use yaml.dump with default_flow_style=False, allow_unicode=True, sort_keys=False
       - Add a YAML comment header: "# Draft glossary — auto-generated. Review and curate before use."

  <action>
    Add a `glossary-gen` subcommand to cli.py following the same pattern as other subcommands:

    ```python
    @cli.command("glossary-gen")
    @click.option("--min-freq", type=int, default=3,
                  help="Minimum term frequency to include (default: 3)")
    @click.option("--output", "-o", type=click.Path(), default=None,
                  help="Write YAML to file instead of stdout")
    @click.pass_context
    def glossary_gen(ctx, min_freq, output):
        """Auto-generate a draft glossary YAML from corpus term frequency."""
        docs_dir = ctx.obj["docs_dir"]

        # Discover and parse all RST files
        conf_py = Path(docs_dir) / "conf.py"
        exclude = load_exclude_patterns(str(conf_py)) if conf_py.exists() else None
        file_list = scan_docs_dir(docs_dir, exclude)

        click.echo(f"Scanning {len(file_list)} RST files for terms...")

        parsed_docs = []
        for filepath, doc_type in file_list:
            content = Path(filepath).read_text(encoding="utf-8", errors="replace")
            parsed = parse_rst(filepath, content, doc_type)
            parsed_docs.append(parsed)

        # Extract, group, generate
        from gauss_doc_qa.glossary_gen import extract_terms, group_terms, generate_glossary_yaml
        term_counts = extract_terms(parsed_docs)
        click.echo(f"Found {len(term_counts)} unique terms")

        groups = group_terms(term_counts, min_freq=min_freq)
        click.echo(f"Grouped into {len(groups)} glossary entries (min frequency: {min_freq})")

        yaml_output = generate_glossary_yaml(groups)

        if output:
            Path(output).write_text(yaml_output)
            click.echo(f"Draft glossary written to {output}")
        else:
            click.echo(yaml_output)
    ```
