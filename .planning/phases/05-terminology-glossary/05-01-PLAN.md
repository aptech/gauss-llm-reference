1. **GlossaryEntry dataclass:**
   ```python
   @dataclass
   class GlossaryEntry:
       canonical: str          # The preferred term (e.g., "GAUSS")
       aliases: list[str]      # Non-canonical variants to flag (e.g., ["Gauss", "gauss"])
       category: str           # Grouping (e.g., "product", "concept", "function")
       description: str = ""   # Optional explanation of the term
   ```

2. **load_glossary(path: str) -> list[GlossaryEntry]:**
   - Read YAML file using `yaml.safe_load`
   - Expected YAML schema:
     ```yaml
     glossary:
       - canonical: "GAUSS"
         aliases: ["Gauss", "gauss"]
         category: "product"
         description: "Always uppercase when referring to the software"
       - canonical: "dataframe"
         aliases: ["data frame", "DataFrame", "data-frame"]
         category: "concept"
     ```
   - Validate: each entry must have `canonical` (non-empty string) and `aliases` (non-empty list of strings)
   - Raise `ValueError` with descriptive message if validation fails (missing fields, wrong types, empty aliases list)
   - Return list of GlossaryEntry

3. **build_alias_map(entries: list[GlossaryEntry]) -> dict[str, GlossaryEntry]:**
   - Build a lookup dict mapping each lowercase alias to its GlossaryEntry
   - This enables O(1) lookup during scanning
   - If two entries share an alias (conflict), raise ValueError naming the conflicting alias and both canonical terms

Dependencies: PyYAML (already in project dependencies -- verify with `pip show pyyaml`)
