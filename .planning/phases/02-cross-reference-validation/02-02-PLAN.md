  **Create `gauss-doc-qa/src/gauss_doc_qa/checkers/links.py` (STRC-03):**

  LinksChecker class with `name = "links"`, `requires_sphinx = True`.

  The `check()` method receives `parsed_doc` (the per-file ParsedDoc) and `**kwargs` containing `sphinx_env` (the Sphinx BuildEnvironment).

  Implementation approach -- use regex on the raw RST source to extract cross-reference targets, then validate against the Sphinx environment registries:

  ```python
  import re
  from gauss_doc_qa.checkers.base import BaseChecker, register_checker
  from gauss_doc_qa.models import ParsedDoc, Finding, Severity

  ROLE_REF_RE = re.compile(r':(\w+):`~?([^`<]+?)(?:\s*<[^>]+>)?`')
  ```

  The checker needs the raw RST content. Read it from `parsed_doc.path` (the file path). For each extracted reference:

  - `:func:` targets: validate against `env.domaindata['gauss']['objects']` using **casefold()** comparison. If not found, produce an ERROR finding with category `"broken_func_ref"`.
  - `:doc:` targets: validate against `env.all_docs` keys. If not found, produce an ERROR finding with category `"broken_doc_ref"`.
  - `:ref:` targets: validate against `env.domaindata.get('std', {}).get('labels', {})`. If not found, produce a WARNING finding with category `"broken_ref"`.
  - Bare backtick references (not preceded by a role): produce an INFO finding with category `"unresolved_bare_ref"` only if they don't resolve against the GAUSS objects registry (casefold). Do NOT use bare backtick detection in this checker -- it is too noisy. Only check explicit role references.

  **Important:** Read raw RST content from `parsed_doc.path` using `Path(parsed_doc.path).read_text()`. The `parsed_doc.raw_doc` is a docutils node which does not contain Sphinx cross-reference nodes.

  Register at module level: `register_checker(LinksChecker())`

  **Create `gauss-doc-qa/src/gauss_doc_qa/checkers/orphans.py` (STRC-04):**

  OrphansChecker class with `name = "orphans"`, `requires_sphinx = True`.

  This checker is **corpus-level**, not per-file. It operates on the entire Sphinx environment to find pages not reachable from the root toctree. The `check()` method should accept a `parsed_doc` argument (to match the BaseChecker interface) but the actual logic works from the `sphinx_env` kwarg.

  Implementation:
  1. Get `root_doc` from `env.config.root_doc` (typically `"index"`)
  2. Walk toctree recursively from root using `env.toctree_includes` to build `reachable` set
  3. Collect `included_fragments` from `env.included` values
  4. For each docname in `env.all_docs`:
     - Skip if in `reachable` set
     - Skip if in `included_fragments`
     - Skip if `'orphan'` key exists in `env.metadata.get(docname, {})`
     - Otherwise, create a WARNING finding with category `"orphan_page"`, file set to `docname + ".rst"`, message `"Page not reachable from any toctree"`

  The checker should store results and only compute once (since it is called per-file but the logic is corpus-wide). Use a class-level `_computed` flag and `_findings` cache. On first call, compute all orphans. On subsequent calls, return only findings matching the current `parsed_doc.path`.

  Register at module level: `register_checker(OrphansChecker())`

  **Create `gauss-doc-qa/src/gauss_doc_qa/checkers/coverage.py` (STRC-07):**

  CoverageChecker class with `name = "coverage"`, `requires_sphinx = True`.

  This checker is **corpus-level** -- it checks whether each Command Reference function name appears in at least one code example anywhere in the docs (including its own page).

  The checker receives via kwargs:
  - `sphinx_env`: the Sphinx BuildEnvironment
  - `all_code_blocks`: `dict[str, list[str]]` mapping docname to list of code block content strings (provided by the CLI runner which collects these during the parse phase)

  Implementation:
  1. Get `gauss_objects = env.domaindata['gauss']['objects']`
  2. Build a single casefold'd corpus string from all code block content across all docs
  3. For each function in gauss_objects where objtype is `'function'`:
     - Check if `funcname.casefold()` appears in the corpus (use word-boundary-aware matching or simple `in` check)
     - If not found, produce a WARNING finding with category `"uncovered_function"`, checker `"coverage"`, message `"Function '{funcname}' has no code example in any doc"`

  Like OrphansChecker, use a `_computed` flag to compute once and cache results. Return only findings matching the current `parsed_doc.path` on subsequent calls.

  Register: `register_checker(CoverageChecker())`

  ---

  **Create `gauss-doc-qa/src/gauss_doc_qa/checkers/seealso.py` (STRC-08):**

  SeeAlsoChecker class with `name = "seealso"`, `requires_sphinx = True`.

  This checker validates that cross-reference targets inside `.. seealso::` directives point to existing pages/functions.

  Implementation:
  1. Read raw RST content from `parsed_doc.path`
  2. Find `.. seealso::` directives using regex: `re.compile(r'\.\.\s+seealso::', re.MULTILINE)`
  3. For each seealso block, extract the content lines (indented lines following the directive)
  4. Extract `:func:` and `:doc:` targets from the seealso content using the same `ROLE_REF_RE` pattern as the links checker: `re.compile(r':(\w+):`~?([^`<]+?)(?:\s*<[^>]+>)?`')`
  5. Validate `:func:` targets against `env.domaindata['gauss']['objects']` with casefold()
  6. Validate `:doc:` targets against `env.all_docs` keys
  7. Produce ERROR findings with category `"broken_seealso_ref"` for unresolved targets

  Register: `register_checker(SeeAlsoChecker())`
