  <action>
  Create the `gauss_doc_qa/frequency/` package with these modules:

  **models.py** -- Data model for frequency results:
  ```python
  @dataclass
  class FunctionFrequency:
      name: str              # function name (e.g. "ols")
      doc_refs: int          # count of :func: cross-references across all RST files
      blog_mentions: int     # count of mentions in aptech.com/blog posts
      combined_score: float  # weighted combined score
      doc_page: str          # docname from domaindata (e.g. "command_ref/ols")
  ```

  **counter.py** -- Cross-reference counting from Sphinx env + RST scanning:
  - `count_crossrefs(docs_dir: str, env) -> dict[str, int]`: Scan ALL RST files in docs_dir for `:func:` role references. Use the same regex pattern as links.py (`ROLE_REF_RE`). For each `:func:` match, normalize the target (strip, casefold) and increment count. Only count references to functions that exist in `env.domaindata['gauss']['objects']` (ignore broken refs). Return dict mapping function_name -> reference_count.
  - Import the regex from links.py or define the same pattern: `re.compile(r":(\w+):`~?([^`<]+?)(?:\s*<[^>]+>)?`")`
  - Walk RST files using `scan_docs_dir()` from `parser.inventory` (same pattern as cli.py scan command).
  - A function's own page referencing itself should NOT count (exclude self-references by checking if the RST file's docname matches the function's doc_page).

  **blog_scraper.py** -- Blog post function mention scraper:
  - `scrape_blog_mentions(known_functions: set[str], max_pages: int = 10) -> dict[str, int]`: Fetch blog listing pages from `https://www.aptech.com/blog/` (paginated). For each blog post URL found on listing pages, fetch the post content. Extract text content, then scan for mentions of any function in `known_functions` (case-insensitive word-boundary match). Return dict mapping function_name -> total_mention_count across all posts.
  - Use `urllib.request` (stdlib, no new dependency needed) for HTTP fetching.
  - Use `html.parser.HTMLParser` (stdlib) to extract text from HTML -- do NOT add beautifulsoup4 dependency.
  - Add a `User-Agent` header: `"gauss-doc-qa/0.1 (documentation-quality-tool)"`.
  - Handle network errors gracefully: if fetching fails, log a warning to stderr and return empty dict. The blog signal is secondary; failure should not crash the tool.
  - Build word-boundary regex from known_functions: `re.compile(r'\b(' + '|'.join(re.escape(f) for f in sorted(known_functions, key=len, reverse=True)) + r')\b', re.IGNORECASE)` -- sort by length descending so longer function names match first.
  - Respect rate limiting: add a 0.5s delay between HTTP requests using `time.sleep(0.5)`.
  - Blog listing page structure: look for `<a>` tags with href containing `/blog/` to find individual post URLs.

  **scorer.py** -- Combine signals into ranked list:
  - `rank_functions(env, doc_ref_counts: dict[str, int], blog_mention_counts: dict[str, int], doc_weight: float = 0.7, blog_weight: float = 0.3) -> list[FunctionFrequency]`:
    - Get all known functions from `env.domaindata['gauss']['objects']`.
    - For each function, compute `combined_score = doc_weight * doc_refs + blog_weight * blog_mentions`.
    - Create `FunctionFrequency` for each function (even those with zero refs -- they still exist in the registry).
    - Sort descending by `combined_score`, then by `doc_refs` as tiebreaker, then alphabetically by `name`.
    - Return the sorted list.

  **__init__.py** -- Package exports:
